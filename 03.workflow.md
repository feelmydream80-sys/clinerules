# 03. General Workflow Rules

모든 작업은 다음 순서를 엄격히 준수한다:

1. 관련 docs/ 문서 확인 (02.documentation.md 참조)
2. **요청 의도 분석 및 확인** (00-core.md 요청 분석 규칙 참조)
   - "새 기능 추가" vs "기존 기능 수정/버그 수정" 구분
   - 에러 메시지 있는 경우: 전체 맥락 확인
   - 모호한 요청: 상세화 요청 후 진행
3. 요구사항과 기존 설계 비교 → 충돌이나 모호함이 있으면 즉시 의문 제기
4. **영향도 분석 수행** (`docs/development/impact-analysis-guide.md` 참조)
   - 변경 대상 모든 사용처 전수 검색
   - 의존성 및 영향 범위 분석
   - 롤백 계획 수립
   - 단계별 배포 계획 수립
5. 요구사항을 잘못 이해한 경우:
   - 즉시 "죄송합니다. 요청을 잘못 이해했습니다."라고 명확히 밝힘
   - 오해한 부분과 근거 문서 설명
   - 올바른 방향 제안
5. 구현 후 테스트 실행 및 구체적 결과 보고

## 지침 변경 후 의무 절차
✅ **모든 지침 문서 변경/추가 후 반드시 수행**:
1.  ✅ 시나리오 작성: 사용자 요청 및 자동 판단 작업에 대한 전체 흐름 시나리오 작성
2.  ✅ 경로 검증: 00-core 나침반에서 올바른 문서로 이동하는지 검증
3.  ✅ 명시적 요청 + 암묵적 자동 작업 케이스 모두 테스트
4.  ✅ 테스트 결과를 해당 문서 하단에 영구 기록
5.  ✅ 문제점 발견시 즉시 수정 후 재테스트
