# 07. 복구 작업 규칙 (Recovery Rules)

> 이 문서는 00-core.md에서 참조합니다.

## 복구 작업 규칙
- "복구" 또는 "롤백" 요청 시 반드시 범위를 확인 후 실행
- 전체 롤백은 명시적으로 "전체 롤백" 또는 "모두 복구" 라고 요청한 경우에만 허용
- 특정 파일/시점 복구 시 영향 범위를 먼저 보고 후 승인받기

## 복구 작업 체크리스트
- [ ] 복구 범위 확인 (전체 vs 부분)
- [ ] 현재 상태 백업 (`git status`, `git log`)
- [ ] 영향 범위 분석 (어떤 파일이 영향을 받는가?)
- [ ] 사용자에게 영향 범위 보고
- [ ] 명시적 승인 받기
- [ ] 복구 실행
- [ ] 복구 후 상태 확인
