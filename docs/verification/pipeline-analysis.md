# 파이프라인 분석 가이드 (Pipeline Analysis Guide)

**Cline 필수 읽기 문서** — 기능 문제 분석 시 반드시 참조

## 1. 목적
- 기능 문제 발생 시 **전체 데이터 파이프라인을 순차적으로 검증**하여 문제 지점을 정확히 특정
- SQL/DB 단에서만 멈추지 않고, 데이터가 화면에 표시되기까지의 **전체 흐름을 추적**

## 2. 분석 순서 (MANDATORY - 순차적 진행)

### 1단계: 백엔드 API 라우트 (routes/)
- 해당 API 엔드포인트가 존재하는가?
- HTTP 메서드(GET/POST)가 맞는가?
- 요청 파라미터를 올바르게 받는가?
- 에러 응답이 적절한가?

### 2단계: 서비스 로직 (service/)
- 비즈니스 로직이 올바른가?
- 데이터 가공/변환이 적절한가?
- 예외 처리가 되어 있는가?

### 3단계: DAO/SQL (dao/, sql/)
- SQL 쿼리가 올바른가?
- 파라미터 바인딩이 적절한가?
- 반환 데이터 구조가 프론트엔드 기대와 일치하는가?
- **NULL 값 처리가 올바른가?** (NOT IN vs NOT EXISTS 주의)

### 4단계: 데이터 정제 함수 (static/js/modules/*/data.js)
- API 응답 데이터를 올바르게 파싱하는가?
- 불필요한 데이터 필터링이 있는가?
- 데이터 변환 로직이 올바른가?

### 5단계: 정렬/렌더링 함수 (static/js/modules/*/table.js, ui.js)
- 정렬 로직이 올바른가? (오름차순/내림차순)
- DOM 렌더링이 올바른가?
- 이벤트 핸들러가 정상 작동하는가?

### 6단계: HTML 템플릿 (templates/)
- 템플릿 구조가 올바른가?
- JavaScript 파일이 올바르게 로드되는가?
- CSS 스타일이 적용되는가?

## 3. 주의사항
- **중간 단계에서 멈추지 말 것**: "SQL이 문제일 것이다"라고 추측하지 말고, 반드시 프론트엔드까지 확인
- **각 단계에서 실제 데이터 확인**: console.log, print, 로그 등으로 실제 데이터 흐름 추적
- **사용자가 문제 위치를 특정해줬으면**: 그 위치를 먼저 확인하고, 사용자가 지시한 분석 순서를 따를 것
- **동일한 실수를 반복하지 말 것**: 사용자가 "아니다", "잘못됐다"고 하면 즉시 중단하고 재분석

## 4. 디버깅 도구
- 백엔드: `print()`, `logging`, Flask debug mode
- 프론트엔드: `console.log()`, 브라우저 개발자 도구 Network 탭
- 데이터베이스: 직접 쿼리 실행하여 결과 확인
