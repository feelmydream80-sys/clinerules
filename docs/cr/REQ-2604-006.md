### 변경 요약 (1줄, 50자 이내)
> 시간 처리 공통함수 적용 및 데이터 처리 규칙 문서화

### 변경 배경 / 이유
> 스케줄 메모 기능에서 시간 표시가 브라우저 시간대 설정에 따라 잘못 표시되는 문제 발생. DB에서 datetime 객체를 가져와 Flask jsonify가 GMT로 변환하면서 프론트엔드 파싱 오류. 이를 해결하고 향후 재발 방지를 위해 데이터 처리 규칙을 문서화함.

### 변경 전 → 변경 후
- Before: DAO에서 datetime 객체 조회 → jsonify가 GMT 변환 → new Date() 파싱 오류
- After : DAO에서 TO_CHAR로 문자열 조회 → 공통함수 formatDBDateTime() 사용

### 영향 범위
> - 스케줄 메모 시간 표시 (collection_schedule.js)
> - 모든 시간/날짜 처리 기능 (dateUtils.js 공통함수)
> - DAO SQL 쿼리 (grp_memo_dao.py)
