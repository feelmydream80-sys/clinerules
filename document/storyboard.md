# MSYS 시스템 스토리보드

## 1. 시스템 개요
- 데이터 수집 및 모니터링 시스템
- Flask 기반 웹 애플리케이션
- PostgreSQL 데이터베이스 사용
- 관리자 대시보드, 데이터 분석, 차트 분석 기능 제공

## 2. 주요 화면 흐름

### 2.1 초기 로그인 페이지
```
1. 사용자 인증
2. 관리자 권한 확인
3. 대시보드로 리다이렉트
```

### 2.2 대시보드 화면
```
1. 날짜 선택 기능
2. 수집 현황 요약
   - 일간 현황
   - 주간 현황
   - 월간 현황
3. Job ID별 상세 현황
4. 이벤트 로그 표시
```

### 2.3 차트 분석 화면
```
1. 데이터 시각화
2. 성공률 추이 차트
3. 장애 코드 분석 차트
4. 사용자 정의 차트
```

### 2.4 데이터 분석 화면
```
1. 원천 데이터 조회
2. 데이터 필터링
3. 통계 분석
4. 추이 분석
```

### 2.5 관리자 화면
```
1. 설정 관리
2. 아이콘 관리
3. Job ID 관리
4. 장애 코드 관리
```

## 3. 주요 기능 흐름

### 3.1 데이터 수집 모니터링
```
1. 사용자가 날짜 범위 선택
2. API 호출로 데이터 조회
3. 대시보드에 시각화
4. 이벤트 로그 표시
```

### 3.2 데이터 분석
```
1. 사용자가 분석 조건 선택
2. API를 통해 데이터 조회
3. 차트 생성 및 표시
4. 결과 저장/다운로드
```

### 3.3 관리자 설정
```
1. 관리자 화면 진입
2. 설정 항목 선택
3. 수정/저장
4. 설정 적용 확인
```

## 4. 데이터 흐름

### 4.1 데이터베이스 구조
```
tb_con_hist
- 수집 이력 저장
tb_con_mst
- 마스터 정보 저장
tb_con_hist_event_log
- 이벤트 로그 저장
```

### 4.2 API 흐름
```
1. 사용자 요청 → Flask 서버
2. 서비스 계층 처리
3. DAO를 통한 DB 접근
4. 결과 반환 및 렌더링
```

## 5. 보안 및 권한 관리
```
1. 사용자 인증
2. 권한 기반 접근 제어
3. API 보안 처리
4. 환경 변수 관리
```

## 6. 시스템 특징
- 모듈화된 아키텍처
- RESTful API 설계
- 데이터 시각화 강화
- 관리자 기능 확장성
- 이벤트 로그 관리
- 데이터 분석 도구 통합
