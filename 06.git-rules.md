# 06. Git 작업 규칙 (Git Rules)

> 이 문서는 00-core.md에서 참조합니다.

## Git 작업 규칙 (CRITICAL - 데이터 손실 방지)
- `git reset --hard`, `git checkout -- .`, `git clean -fd` 등 **되돌릴 수 없는 명령어**는 절대 사용자 명시적 승인 없이 실행 금지
- Git 명령어 실행 전 반드시:
  1. **어떤 저장소에서 실행되는지 확인** (root 저장소 vs 서브폴더 별도 저장소 vs 서브모듈)
  2. 현재 상태(`git status`)와 변경 가능한 파일 목록 확인
  3. 사용자에게 **"이 명령어는 [영향]을 미치며, 되돌리려면 [방법]이 필요합니다. 진행하시겠습니까?"** 확인
- 로컬 커밋이 원격에 push되지 않은 경우, `git reset --hard`는 **로컬 작업물을 영구 삭제**할 수 있음
- 서브폴더가 별도의 Git 저장소인지 확인 방법: `cd 폴더명 && git remote -v`로 원격 저장소 URL 확인
- **사용자가 "XX 폴더에서 git pull 해줘"라고 요청 시:**
  1. 해당 폴더가 별도의 Git 저장소인지 확인
  2. 해당 폴더의 remote URL이 요청 의도와 일치하는지 확인
  3. root 저장소의 remote와 혼동하지 않도록 주의

## 커밋 전 필수 확인 (00-core.md 기반)

> ⚠️ **커밋 요청 시 반드시 아래 순서로 확인**

1. [ ] **01.legacy-protection.md** - 레거시 코드 보호 확인
2. [ ] **00-core.md** - 작업 유형 분류 확인
3. [ ] 변경 유형 판별 - **기능 작업** / **비기능 작업**

### 작업 유형 구분

| 구분 | 조건 | FP |
|------|------|-----|
| **기능 작업** | feat, fix, refactor, docs | 산출 가능 (N) |
| **비기능 작업** | inquiry, analysis, documentation, review | **산출 불가 (-)** |

### 비기능 작업 정의

| 유형 | 설명 | 예시 |
|------|------|------|
| **inquiry** | 질문/문의 | "기본값이 뭐야?", "어떤 파일이야?" |
| **analysis** | 분석/디버깅 | "데이터가 안 떠", "에러 발생" |
| **documentation** | 문서 확인/작성 | "문서 만들어줘", "설명해줘" |
| **review** | 코드/설계 리뷰 | "코드 리뷰 해줘" |

## CR 보고서 생성 (a.txt 기반)

> ⚠️ **커밋 요청 시마다 CR 보고서 생성**

### CR ID 생성 규칙

- 형식: `REQ-yymm-nnn`
- 예시: `REQ-2604-001` (2026년 4월 첫 번째 요청)

### 커밋 메시지 형식 (a.txt 기반)

```
REQ-yymm-nnn: <변경 요약 50자 이내>

- 변경 상세 내용 1
- 변경 상세 내용 2

Refs: REQ-yymm-nnn
FP: N | 공수: NH | 작업유형: 기능/비기능
```

### 보고서 저장 위치

```
.clinerules/docs/cr/
└── REQ-yymm-nnn.md
```

### 보고서 템플릿

