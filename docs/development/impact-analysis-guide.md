# 영향도 분석 가이드 (Impact Analysis Guide)

**모든 기능 변경/개선 작업 전 필수 수행 항목**

## 1. 목적
기능 변경 또는 개선 작업 수행 전 전체 시스템에 미치는 영향을 정확히 분석하여 예기치 않은 부작용을 사전에 방지한다.

## 2. 표준 절차 (MANDATORY - 모든 작업 전 필수 실행)

### 2.1 사용처 전수 검색
1. 변경 대상 필드, 함수, API, 테이블에 대한 정규식 검색 실행
2. 모든 관련 파일과 코드 라인 식별
3. 호출 계층 구조 분석
4. 직/간접 의존성 확인

### 2.2 의존성 그래프 작성
1. 영향 받는 컴포넌트 목록 작성
2. 호출 흐름 다이어그램 작성
3. 데이터 흐름 추적
4. 외부 시스템 연동점 확인

### 2.3 영향 범위 분류
| 등급 | 설명 | 조치 |
|------|------|------|
| 치명적 | 핵심 기능 장애, 데이터 손실 가능성 | 작업 중단, 별도 승인 절차 |
| 중요 | 주요 기능 영향, 성능 저하 | 단계별 배포, 롤백 계획 필수 |
| 경미 | UI 스타일, 비핵심 기능 | 일반 배포 절차 |

### 2.4 회귀 테스트 시나리오 작성
1. 기존 정상 동작 기능 목록 추출
2. 엣지 케이스 테스트 항목 작성
3. 성능 영향 검증 항목
4. 데이터 정합성 검증 항목

### 2.5 롤백 계획 수립
1. 롤백 방법과 절차 문서화
2. 복구 시나리오 작성
3. 데이터 백업 계획
4. 롤백 확인 절차

### 2.6 단계별 배포 계획
1. 변경 사항을 최소 단위로 분할
2. 각 단계별 검증 기준 설정
3. 모니터링 항목 정의
4. 롤아웃 일정 계획

## 3. 검증 체크리스트
- [ ] 모든 사용처를 검색했는가?
- [ ] 의존성을 모두 식별했는가?
- [ ] 영향 등급을 적절히 분류했는가?
- [ ] 회귀 테스트 케이스를 작성했는가?
- [ ] 롤백 계획을 수립했는가?
- [ ] 단계별 배포 계획이 있는가?

## 4. 위반시 조치
위 절차를 준수하지 않고 작업을 진행한 경우 즉시 작업을 중단하고 절차를 준수할 때까지 진행 불가.
