# 04. 폴더 명칭 규칙

> 문서를 분리할 때 폴더 명칭은 **나침반의 역할을 하는 문서의 이름**을 따름.

## 올바른 예시

**분리 전:**
```
docs/msys/
├── core-admin-page-rules.md    ❌ 파일명에 접두사 사용
└── core-design-change-workflow.md
```

**分离 후 (올바름):**
```
docs/msys/
└── core/
    ├── admin-page-rules.md
    └── design-change-workflow.md
core.md                         ← 나침반 (참조만)
```

## 핵심 원칙

1. **나침반 문서명 = 폴더명**: 분리된 문서를 담는 폴더는 나침반 문서의 이름과 같음
2. **소문자 사용**: 폴더명과 파일명은 소문자 사용
3. **케밥 케이스**: 단어 구분은 하이픈(`-`) 사용

## 폴더 생성 절차

1. 나침반 문서명 확인 (예: `core.md`)
2. 나침반 문서명에서 `.md` 제거 → 폴더명 (예: `core/`)
3. 분리된 문서들을 해당 폴더로 이동
4. 나침본 문서에 하위 문서 목록 추가

## 잘못된 예시 (⚠️ 주의)

```
docs/msys/
├── msys-core/           ❌ msys 접두사 불필요
│   └── admin-page-rules.md
└── msys-design/        ❌ msys 접두사 불필요
    └── design-change-workflow.md
```

> **핵심**: 폴더 명칭은 나침반 문서명에서 `.md`를 제거한 것. 프로젝트 접두사는 불필요.
