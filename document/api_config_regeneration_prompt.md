# 프롬프트: 전체 소스 코드 분석을 통한 `apiConfig.js` 파일 자동 생성

## 목표

현재 Flask 프로젝트의 전체 소스 코드를 분석하여, 프론트엔드에서 사용하는 `static/js/modules/common/apiConfig.js` 파일을 최신 API 엔드포인트 정보로 자동 재생성합니다. 이 파일은 API 테스트 페이지의 동적 UI 구성에 사용됩니다.

## 수행 과정

다음과 같은 단계별 계획에 따라 작업을 수행합니다.

### 1단계: 전체 API 엔드포인트 식별

- **작업:** `routes` 디렉토리 내의 모든 `*_routes.py` 파일을 분석합니다.
- **세부 내용:**
    - 각 파일에서 Flask의 `@*.route()` 데코레이터를 찾아 등록된 모든 API 엔드포인트(URL 경로)를 식별합니다.
    - 각 엔드포인트에 지정된 HTTP 메소드(예: `GET`, `POST`, `PUT`, `DELETE`)를 파악합니다.
    - 해당 엔드포인트와 연결된 컨트롤러 함수 이름을 기록합니다.

### 2단계: 각 엔드포인트의 요청 파라미터 및 데이터 구조 분석

- **작업:** 1단계에서 식별된 각 컨트롤러 함수의 로직을 심층적으로 분석하여 필요한 요청 파라미터를 파악합니다.
- **세부 내용:**
    - **요청 데이터:** 함수 내에서 `request.args`, `request.form`, `request.json` 등을 통해 어떤 데이터를 읽어오는지 확인하여 필수 및 선택 파라미터를 식별합니다.
    - **서비스 계층 분석:** 컨트롤러 함수가 호출하는 `service` 계층의 메소드를 따라갑니다.
    - **매퍼 및 DAO 계층 분석:** `service` 계층에서 호출하는 `mapper` 또는 `dao` 계층의 코드를 분석하여, 데이터베이스와 상호작용하는 데 필요한 최종 데이터 모델과 구조를 파악합니다.
    - **데이터 흐름 종합:** `routes` -> `service` -> `mapper`/`dao`로 이어지는 전체 데이터 흐름을 종합하여 각 API가 요구하는 최종적인 데이터의 형태(key, value, 자료형 등)를 명확히 정의합니다.

### 3단계: `apiConfig.js` 파일 재생성

- **작업:** 1, 2단계의 분석 결과를 바탕으로 `static/js/modules/common/apiConfig.js` 파일을 새로 작성합니다.
- **세부 내용:**
    - 기존 파일의 `apiGroups` 객체 구조를 유지합니다.
    - 각 API 엔드포인트를 기능별로 적절한 그룹(예: "관리자 설정", "사용자 관리", "분석")에 배치합니다.
    - 각 API에 대해 다음 정보를 포함하는 객체를 생성합니다.
        - `title`: API의 기능을 명확하게 설명하는 한글 제목 (예: "모든 관리자 설정 조회")
        - `endpoint`: 식별된 API의 URL 경로 (예: `/api/admin/settings/all`)
        - `method`: 식별된 HTTP 메소드 (예: `GET`, `POST`)
        - `defaultData`: API 테스트에 즉시 사용할 수 있는 기본 요청 데이터 예시. 2단계에서 분석한 파라미터 구조를 바탕으로 구체적인 값을 포함하여 작성합니다. (예: `{"user_id": "testuser"}`)
    - `write_to_file` 도구를 사용하여 분석된 내용으로 `apiConfig.js` 파일의 전체 내용을 덮어씁니다.

## 최종 결과물

- 최신 API 명세가 완벽하게 반영된 `static/js/modules/common/apiConfig.js` 파일.
- 이 프롬프트 자체 (`document/api_config_regeneration_prompt.md`)는 향후 동일한 작업을 수행해야 할 다른 개발자를 위한 가이드 역할을 합니다.
