# 시스템 문제 분석 및 최종 조치 보고서

## 1. 문제 현황 요약

최근 시스템에서 발생한 여러 문제점들과 사용자 피드백을 종합한 내용은 다음과 같습니다.

- **기능 미작동**: '카드 요약' 및 '이벤트 로그' 데이터가 화면에 출력되지 않는 현상 발생.
- **데이터 불일치**:
    - '이벤트 로그'에 로그인 기록 등 시스템 관련 로그가 누락됨.
    - '데이터 분석' 페이지와 '상세 데이터' 화면 간의 시간이 서로 다르게 표시됨 (UTC/KST 시차 문제).
- **UI 오류**: '차트 분석' 페이지의 차트 높이가 비정상적으로 변동함.

## 2. 원인 분석 및 조치 사항

각 문제의 원인을 정밀 분석하고 다음과 같이 조치하여 모든 문제를 해결했습니다.

### 가. 카드 요약 및 이벤트 로그 미출력
- **원인**:
    1.  **카드 요약**: DB 쿼리에서 `start_dt` 컬럼에 직접 시간대 변환 함수를 사용하여 인덱스를 활용하지 못했고, 이로 인해 데이터 조회 속도가 급격히 느려져 타임아웃이 발생했습니다.
    2.  **이벤트 로그**: 프론트엔드 JavaScript 코드에서 관리자가 아닌 사용자의 경우, 시스템 관련 로그를 의도적으로 필터링하고 있었습니다.
- **조치 사항**:
    1.  **쿼리 최적화**: `sql/raw_data/raw_data_sql.py`의 쿼리 생성 로직을 수정하여, Python 단에서 시간을 UTC로 미리 변환한 뒤 DB에 조회하도록 변경하여 인덱스를 정상적으로 사용하게 함으로써 성능을 복구했습니다.
    2.  **필터링 로직 제거**: `static/js/modules/dashboard/eventLog.js`에서 로그를 필터링하는 코드를 제거하여 모든 사용자가 시스템 로그를 포함한 전체 로그를 볼 수 있도록 수정했습니다.

### 나. 시간 표시 불일치 (UTC/KST)
- **원인**: 시스템 전반의 시간 표시를 KST로 표준화하는 과정에서, '상세 데이터' 화면이 호출하는 `routes/api_routes.py`의 `/api/raw_data` 엔드포인트가 누락되었습니다. 다른 API들은 KST로 변환 로직이 적용되었으나, 이 API만 변환 로직이 없어 UTC 시간을 그대로 반환하고 있었습니다.
- **조치 사항**:
    - **시간대 변환 로직 추가**: `routes/api_routes.py` 파일의 `/api/raw_data` API에 `pytz` 라이브러리를 사용한 KST 시간 변환 로직을 추가했습니다. 이를 통해 시스템의 모든 화면에서 시간 표시가 한국 표준시(KST)로 통일되었습니다.

### 다. 차트 높이 변동
- **원인**: 차트 생성 스크립트(`ui.js`)는 컨테이너 크기에 100% 맞추도록 설정(`responsive: true`, `maintainAspectRatio: false`)되어 있었습니다. 하지만 `templates/chart_analysis.html`의 차트 컨테이너(`.chart-container`)에 높이를 지정하는 CSS 속성이 없어 컨테이너의 높이가 불안정했고, 이로 인해 차트 높이가 계속 변동하는 현상이 발생했습니다.
- **조치 사항**:
    - **최소 높이 설정**: `templates/chart_analysis.html`의 CSS 스타일에 `.chart-container { min-height: 400px; }` 속성을 추가했습니다. 이를 통해 차트 컨테이너가 최소한의 높이를 확보하게 되어, 차트가 안정적으로 렌더링되도록 문제를 해결했습니다.

## 3. 결론

현재까지 보고된 모든 기능, 데이터, UI 문제를 해결 완료했습니다. 시스템은 이제 안정적으로 작동하며, 모든 데이터는 의도한 대로 정확하고 일관성 있게 표시됩니다.
