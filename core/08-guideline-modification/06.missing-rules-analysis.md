# 06. 누락된 규칙 분석 및 새로운 메인 지침 추가 절차

> 이 문서는 00-core.md에 누락된 규칙을 분석하고, 새로운 메인 공통 지침을 추가할 때의 절차를 정의합니다.

---

## 1. 00-core.md에 누락된 규칙 분석

### 누락된 규칙 목록

| # | 누락된 규칙 | 결과 |
|---|------------|------|
| 1 | 새로운 **메인 공통 지침** 추가 시 core/ 번호 할당 방법 | 어떤 번호를 붙여야 하는지 몰랐음 |
| 2 | 기존 다른 폴더(files/, docs/ 등)의 파일들을 core/로 이동하는 절차 | 이동 절차 예측 불가 |
| 3 | 폴더 명칭 규칙의 00-core.md 참조 | 04.folder-naming.md가 00-core에서 링크 안 됨 |
| 4 | 수정 후 필수 절차 (post-modification) | 테스트 결과 기록이 자동 연결 안 됨 |

### 원인 분석

1. **00-core.md가 "나침반" 역할만 하도록 설계** → 구체적 절차는 하위 문서에 위임
2. **08-guideline-modification이 지침 수정에만 초점** → 새 지침 추가 절차는 불분명
3. **폴더 명칭 규칙(04.folder-naming.md)이 00-core.md에 링크되지 않음** → 참조 누락

## 2. 새로운 메인 공통 지침 추가 시 번호 할당 규칙

### 번호 체계

| 번호 범위 | 용도 |
|----------|------|
| 00 | Core Rules (나침반) |
| 01-09 | 기존 메인 공통 지침 |
| 10-19 | 분석/나침반 관련 |
| 20-29 | (예약) |
| 30~ | 새로운 메인 지침 순차 할당 |

### 번호 할당 원칙

1. **기존 번호 확인**: Glob으로 현재 core/ 폴더 내 파일 번호 확인
2. **최소 미사용 번호 선택**: 빈 번호가 있으면 해당 번호 사용
3. **연속 번호优先**: 관련 지침은 연속된 번호 사용 권장
4. **00-core.md 동시 업데이트**: 분류표와 문서 위치표에 필수 추가

### 예시

```
기존: 01~10 사용 중
새로 추가: 11, 12, 13 순차 할당
```

## 3. 기존 폴더의 파일들을 core/로 이동하는 절차

### 이동 시 고려사항

| 항목 | 확인 사항 |
|------|----------|
| 파일 존재 여부 | Glob으로 실제 파일 경로 확인 |
| 메인 공통 지침 여부 | 여러 프로젝트에서 공통으로 사용되는 지침인가? |
| 번호 중복 | core/ 내 기존 번호와 충돌 여부 확인 |
| 참조 링크 | 이동 후 링크 업데이트 필요 여부 |

### 이동 절차

```
1. 파일 존재 여부 확인 (Glob)
2. 메인 공통 지침 판단
3. 번호 할당 (연속 번호 확인)
4. 새 위치에 파일 생성
5. 00-core.md 분류표/문서 위치표 업데이트
6. 기존 폴더 파일 삭제
7. 테스트 결과 기록 (05.post-modification.md)
```

## 4. 폴더 명칭 규칙의 00-core.md 참조

### 현재 상태

- 04.folder-naming.md: 존재함

### 해결 방안

```

```

## 5. 수정 후 필수 절차 (현재 상태)

### 05.post-modification.md 확인

- 테스트 결과 기록 ✅ 존재
- 00-core.md 경로 검증 ✅ 존재

### 개선 제안

00-core.md에 "지침 추가/삭제/수정" 항목에서 05.post-modification.md로의 링크를 명확히 표시:

```

→ [수정 후 절차](core/08-guideline-modification/05.post-modification.md) |
```

## 6. 종합 체크리스트

새로운 메인 공통 지침 추가 시 반드시 수행:

- [ ] 1. 기존 파일 이동: Glob으로 파일 존재 여부 확인
- [ ] 2. 번호 할당: core/ 내 최소 미사용 번호 선택
- [ ] 3. 파일 생성: 새 위치에 파일 작성
- [ ] 4. 00-core.md 업데이트: 분류표 + 문서 위치표 동시 수정
- [ ] 5. 기존 파일 삭제: 원본 폴더에서 파일 제거
- [ ] 6. 테스트 기록: 05.post-modification.md에 결과 기록
- [ ] 7. 참조 검증: 새 링크가 정상 작동하는지 확인
