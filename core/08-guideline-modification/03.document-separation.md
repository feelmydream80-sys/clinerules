# 03. 문서 분리 기준

## 문서 분리 기준 (80줄 초과 시 → 주제 응집성 기준으로 판단)

> 단순히 줄 수가 80줄을 넘는다고 분리하지 않는다.
> **같은 파일에 서로 다른 작업 유형의 규칙이 혼재**할 때 분리를 고려한다.

### 분리가 필요한 경우
- 한 파일에 "UI 작업 규칙"과 "API 작업 규칙"이 함께 있는 경우
- 한 파일에 "절차 규칙"과 "예시/시나리오"가 섞여 파악이 어려운 경우

### 분리 절차
1. 분리 계획(어떤 파일을 어떻게 나눌지)을 사용자에게 **먼저 제시**
2. 사용자 승인 후 실행
3. 기존 문서는 나침반 역할로 유지 (내용 삭제, 참조 경로 추가)
4. 00-core.md 분류표 및 문서 위치 표 업데이트

### 분리 형식
```
.clinerules/
└── {기존 문서명}/         ← 폴더 생성
    ├── {주제A}.md
    └── {주제B}.md
{기존 문서명}.md           ← 나침반으로 유지 (내용 → 참조 경로로 대체)
```

---

### 00-core.md 분리 시 (최우선 적용)

> 00-core.md는 프로젝트의 나침반으로, 다른 문서와 구분하여 특별 취급

1. `.clinerules/core/00-core/` 폴더 생성
2. 00-core.md는 나침반으로 `.clinerules/00-core.md` (root)에 유지
3. 상세 내용은 `core/00-core/` 폴더에 주제별로 자연스럽게 번호 증가:
   - 예: `01.global-rules.md`, `02.triggers.md`, `03.plan-mode.md`
4. 기존 문서(01.legacy-protection.md 등)는 `.clinerules/` root에 유지
5. 00-core.md (root)에서 새 폴더 경로로 안내
