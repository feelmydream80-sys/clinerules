### 변경 요약 (1줄, 50자 이내)
> 팝업(공지) 관리 시스템 구현 - 관리자 설정 및 사용자 표시 기능

### 변경 배경 / 이유
> 사용자 요청:
> 1. 관리자가 팝업을 생성/수정/삭제할 수 있는 기능 필요
> 2. 이미지 첨부 가능한 팝업 지원
> 3. "하루동안 보지 않기" 기능 필요
> 4. 팝업 표시 기간 관리자 설정 필요
> 5. 다중 팝업 동시 표시 지원

### 변경 전 → 변경 후
- Before: 팝업 시스템 없음
- After : 
  - TB_POPUP_MST 테이블 생성
  - 관리자 페이지에 "팝업 관리" 탭 추가
  - 팝업 CRUD API 구현
  - 사용자 화면 팝업 표시 모듈 구현
  - "하루동안 보지 않기" 기능 (localStorage)

### 영향 범위
> 1. 데이터베이스: TB_POPUP_MST 테이블 신규 생성
> 2. Backend: DAO, Service, Routes 3개 파일 추가
> 3. Frontend: 
>    - 팝업 관리 탭 모듈 (popupManagement.js)
>    - 팝업 표시 모듈 (popupManager.js, popupStorage.js)
>    - 팝업 스타일 (popup.css)
> 4. Templates: base.html (팝업 모듈 로드), mngr_sett.html (탭 추가)

**[상세 내용]**
> 팝업(공지사항) 관리 시스템을 구현했습니다. 관리자는 관리자 설정 페이지에서 팝업을 생성하고, 이미지를 첨부하며, 표시 기간을 설정할 수 있습니다. 사용자는 웹사이트 접속 시 팝업을 확인하고 "하루동안 보지 않기"를 선택할 수 있습니다.

> - 팝업 CRUD (제목, 내용, 이미지, 링크, 기간 설정)

> - 다중 팝업 순차 표시
> - 반응형 디자인 (모바일/데스크톱)
