# 오류 노트

**날짜 및 시간:** 2025-11-06 19:57

**오류 현상:**
'admin' 사용자로 로그인했음에도 불구하고, 관리자 페이지('/admin/*')에 접근 시 "접근권 없음" 페이지 또는 대시보드로 리디렉션되는 문제가 발생함. 데이터베이스에는 해당 사용자에게 관리자 권한('mngr_sett')이 정상적으로 부여되어 있었음.

**오류 원인:**
문제는 각 라우트의 `@admin_required` 데코레이터가 아니었습니다. Flask 애플리케이션의 메인 파일인 `msys_app.py`에 정의된 전역 `before_request` 핸들러에 오래된 권한 확인 로직이 남아있었습니다.

이 핸들러는 `/admin`으로 시작하는 모든 요청을 가로채, 세션에 `'admin'`이라는 권한이 있는지 확인했습니다. 하지만 시스템은 `'is_admin'` 컬럼을 제거하고 `'mngr_sett'` 권한을 사용하도록 리팩토링된 상태였습니다. 따라서 이 전역 핸들러가 항상 권한이 없다고 잘못 판단하여, 실제 엔드포인트의 `@admin_required` 데코레이터가 실행되기도 전에 접근을 차단하고 있었습니다.

**해결 방법:**
`msys_app.py` 파일의 `before_request` 핸들러 내부에 있는 관리자 경로 접근 확인 로직을 다음과 같이 수정했습니다.

- **기존 코드:**
  ```python
  if 'admin' not in g.user.get('permissions', []):
  ```

- **수정된 코드:**
  ```python
  if 'mngr_sett' not in g.user.get('permissions', []):
  ```

이 수정을 통해 애플리케이션 전역에서 사용되는 권한 확인 로직을 최신 기준('mngr_sett')으로 통일하여 문제를 해결했습니다.
