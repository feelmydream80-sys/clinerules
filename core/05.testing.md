# 05. Testing and Reporting Rules

## 테스트 범위 원칙

> ⚠️ **"모든 기능 + Edge Case"를 매번 전수 테스트하지 않는다.**
> 수정 범위에 비례해서 테스트 범위를 결정한다.

| 수정 범위 | 테스트 범위 |
|----------|------------|
| 단일 함수 / 단일 요소 수정 | 해당 기능만 테스트 |
| 모듈 단위 수정 | 해당 모듈 + 직접 연관 기능 테스트 |
| 공통 모듈 / 공통 컴포넌트 수정 | 해당 모듈을 사용하는 모든 기능 테스트 |
| 전체 리팩토링 / 아키텍처 변경 | 전체 기능 + Edge Case 테스트 |

---

## 테스트 후 보고 규칙

코드 수정 후 반드시 다음을 준수:

- 테스트 결과는 모호하게 "완료"라고 하지 말고, **기능별로 구체적으로 보고**
  ```
  ✅ {기능명}: 정상 동작
  ⚠️ {기능명}: {문제 내용} (재현 방법: {방법})
  ❌ {기능명}: {실패 내용}
  ```
- 사용자에게 Flask 앱 실행 방법과 접속 URL 명확히 안내
  (예: `cd wordcloud_project && python web/app.py 실행 후 http://localhost:5001/stopwords 확인`)

---

## Edge Case 테스트 기준

공통 모듈 수정 또는 전체 리팩토링 시에만 Edge Case를 포함한다:
- 빈 값 / null 입력
- 최대값 / 최소값 경계
- 네트워크 오류 / 타임아웃
- 권한 없는 접근

단순 버그 수정이나 단일 요소 수정 시에는 Edge Case 테스트를 생략해도 된다.
