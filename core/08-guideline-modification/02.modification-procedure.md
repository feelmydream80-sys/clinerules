# 02. 수정 요청 시 절차

## ⛔ 최우선 잠금 규칙

**`.clinerules/` 내 파일은 사용자가 명시적으로 "지침 수정해줘", "규칙 바꿔줘", "clinerules 업데이트해줘"라고 요청한 경우에만 수정한다.**

- 지침이 outdated처럼 보여도 자동으로 수정하지 않는다
- 지침과 충돌하는 상황이 발생해도 지침 파일을 수정하지 않고, 사용자에게 상황을 보고한다
- 코드 작업 중 지침의 문제점을 발견하면 → 작업 완료 후 "지침 XX.md의 YY 항목이 현재 상황과 맞지 않습니다. 수정할까요?"로 알림

---

### 새 규칙 추가 시
1. 00-core.md의 핵심 규칙 영역에 추가
2. 기존 규칙과 중복되지 않도록 확인
   - 동일 취지 규칙이 이미 있으면: "XX.md에 유사한 규칙이 있습니다. 해당 문서에 추가할까요, 아니면 별도로 작성할까요?" 확인
3. 추가 후 00-core.md의 분류표 및 문서 위치 표도 함께 업데이트

### 규칙 삭제 시
1. 삭제 전 해당 규칙을 참조하는 다른 문서 확인
2. 참조 문서가 있으면 사용자에게 알리고 일괄 처리 여부 확인
3. 삭제 후 00-core.md 링크 정리

### 규칙 수정 시
1. 수정 전 현재 내용과 수정 후 내용을 Before/After로 사용자에게 제시
2. 승인 후 수정

## 참조 검증 절차 (수정/추가 전)

1. **Glob으로 참조하는 모든 문서 존재 여부 확인**
2. 존재하지 않으면 → 사용자에게 "XX 문서가 없습니다. 생성할까요?"
3. **새 프로젝트 문서 생성 시 README.md 필수 항목**:
   - 프로젝트 개요
   - 작업 유형별 문서 위치
   - 서비스/모듈 경로
