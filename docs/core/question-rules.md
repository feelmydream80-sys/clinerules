# 요청 불확실 시 질문 규칙 (Question Rules)

> 이 문서는 00-core.md에서 참조합니다.
> **나침반으로서**: 추측 대신 구체적인 문서/파일 경로를 먼저 안내하세요.

## 요청 불확실 시 질문 규칙 (MANDATORY)

- 사용자 요청을 정확히 파악하지 못하면 **즉시 중단**하고 사용자에게 질문
- **추측해서 진행 금지** - 나침반으로서 먼저 분석할 문서가 있는지 확인

## 올바른 진행 방식

### 1단계: 나침반 역할 수행
- 요청을 분석하여 어떤 유형의 작업인지 파악
- **00-core.md의 작업 유형 분류표** 참조하여 이동할 문서确定

### 2단계: 구체적 파일 안내
-提问하기 전에 관련 파일 최소 1곳 이상 분석
- "이 문제를 분석하려면 다음 파일들을 확인하세요" 명시

### 3단계: 질문 (필요시)
- 분석 후에도 모르면 사용자에게 질문
- 질문 형식:
  ```
  "요청을 명확히 파악하기 위해 [파일명]을 확인했습니다.
   이 작업은 [작업 유형]으로 분류되며,
   [문서명]을 참조해야 할 것 같습니다.
   맞습니까? 아니면 다른 문서를 봐야 합니까?"
  ```

## 질문 전 체크리스트

- [ ] 00-core.md에서 작업 유형 분류 확인
- [ ] 해당 문서 읽기
- [ ] 관련 파일 최소 1곳 분석
- [ ] 그래도 모르면 질문

## 시나리오 테스트

### 시나리오: 워드클라우드 생성 안 되는 문제

**상황**: 사용자가 분석 시작 버튼 누르면 워드클라우드가 안 만들어짐 보고

**올바른 진행**:
1. 00-core.md → "기능 문제 분석/디버깅" 확인
2. 03.workflow.md + docs/verification/pipeline-analysis.md 참조
3. 관련 파일 분석: api_routes.py, wordcloud_generator.py
4. 문제 원인 파악 후 수정

**결과**: ✅ 질문 없이 문제 해결 가능 - 나침반 역할 수행

### 시나리오: 새로운 기능 추가 요청

**상황**: 사용자: "사용자 피드백 기능 추가해줘"

**올바른 진행**:
1. 00-core.md → "새 기능 추가" 해당하는지 확인

4. 기존 화면 분석: docs/ui/common/screen-domain.md
5. 구현 후 테스트

**결과**: ✅ 나침반이 정확한 방향 안내
