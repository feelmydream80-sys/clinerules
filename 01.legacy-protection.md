# 01. Legacy & Stable Code Protection (CRITICAL - 절대 위반 불가)

## 핵심 원칙 (모든 작업에 최우선 적용)
**사용자가 명시적으로 "수정해", "리팩토링해", "개선해", "바꿔줘"라고 요청하지 않은 코드는 절대 수정하지 마라.**

정상적으로 동작하고 있는 기존 코드를 AI가 스스로 판단해서 건드리는 행위를 철저히 금지한다.

## 보호 대상
- 현재 정상적으로 동작하는 모든 기능과 모듈
- 사용자가 "legacy", "기존 코드", "손대지 마", "그대로 유지해"라고 언급한 영역
- `/legacy/` 폴더 전체와 그 하위 파일
- 파일명에 `_legacy`, `old_`, `deprecated_`, `stable_` 등이 포함된 파일
- `docs/legacy/` 또는 `archive/` 폴더

## 엄격한 금지 규칙
- 사용자가 **명시적으로 수정 요청**하지 않은 파일은 **어떤 부분도 수정, 삭제, 리팩토링, 최적화, 스타일 변경, 리팩토링, 코드 정리 금지**
- "더 좋게 만들자", "현대화하자", "클린 코드로 바꾸자" 같은 판단으로 기존 코드를 건드리지 마라
- 보호 대상 코드는 **읽기만** 허용
- 기존 코드를 복사해서 수정하거나, 기반으로 개선하려는 시도 절대 금지  
  → 새로운 기능이나 변경이 필요하면 **완전히 새로운 파일이나 새로운 모듈**에 작성

## 새 작업 원칙
- 새로운 기능 추가, 버그 수정, 새로운 요구사항 구현 시에도  
  **기존 정상 동작 코드를 최대한 건드리지 말고**, 필요한 부분만 최소한으로 수정하거나 새로 작성
- "개선"이나 "리팩토링"은 사용자가 **구체적으로 요청**한 경우에만 허용

## 예외 처리 절차 (반드시 준수)
기존 정상 동작 코드를 수정해야 할 이유가 있다고 판단되면:
1. 작업을 즉시 중단한다.
2. 정확한 파일 경로와 "왜 수정해야 하는지", "어떻게 수정할 계획인지"를 사용자에게 상세히 보고한다.
3. **사용자의 명시적인 승인** ("이 파일 수정 OK", "리팩토링 허용" 등)을 받은 후에만 진행한다.
4. 승인 없이 절대 수정하지 않는다.

이 규칙을 위반하면 정상 동작하던 기능이 망가질 위험이 매우 크다. 항상 가장 보수적으로 행동하라.
